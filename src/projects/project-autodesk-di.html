Autodesk Fusion: Achieving Device Independence

Transforming a desktop-only CAD application into a unified, cross-platform experience that scales seamlessly across all devices

Project Summary

Over three years, I led the UX architecture for transforming Autodesk Fusion from a desktop-only application into a device-independent platform that works seamlessly across mobile, tablet, web, and desktop environments.

Role

UX Architect (iOS/Web)

Duration

3+ Years

Team

2 Senior UX Designers, 2 User Researchers, Design Manager, Multiple Dev Teams

Platform

Cross-platform (Mobile, Tablet, Web, Desktop)

The Challenge

"How might we design a unified responsive experience that scales across all platforms?"

Fusion faced increasing pressure from competitors who already had web, tablet, and phone apps. Device availability is becoming the default expectation for any modern product, and mobilizing Fusion had the potential to grow adoption and sales significantly.

Understanding the Problem

What is Fusion?

Autodesk Fusion helps users manufacture products, supporting the end-to-end workflow. Users range from large mining equipment manufacturers to footwear designers. While best known for its 3D design environment, Fusion has acquired many technologies over the years, adding workspaces like simulation, generative design, electronics, and drawings. It also has a robust data management system.

In short: it's complex, has design consistency issues, and features a dense UI optimized exclusively for desktop use.

Competitive Pressure

Competitors already had web, tablet, and phone apps while Fusion remained desktop-only, risking subscription loss in 5-10 years.

Complex Dense UI

Fusion's interface was optimized solely for mouse and keyboard interactions, limiting usability across other devices and contexts.

No Cross-Platform Strategy

There was no established framework to extend Fusion's reach across devices or accommodate diverse input methods.

Modern Expectations

Availability on any device is becoming the default expectation for modern software products.

Strategic Approach

This project required a dual strategy: developing capabilities guidelines for what workflows and data are available on each device, and creating new interaction concepts for cross-platform experiences.

Phase 1

Capabilities Strategy

Guidelines & rules for what workflows and data are available on what devices & contexts

User Value Framework

Working with our user researcher, we created a Practitioner Value Framework based on comprehensive user surveys. This revealed that to drive adoption, we needed to focus on features/functionality and deliver an exceptional user experience.

Customer Workshop at Autodesk University

I ran a customer-focused workshop with 20 users to gain deeper understanding of specific needs on each device type.

Key Finding: "Customers want to perform existing tasks with more mobility."

Phase 2

Interaction Strategy

Concepts for new workflows and components optimized for different devices and input methods

Device-Specific User Insights

  • 📱 Phone: Mostly used for sharing, quick edits, and quick annotations. Users will edit "in a pinch"
  • 📊 Tablet: High demand for modeling, annotation, and CNC updates. Most 'authoring' personas want Fusion on tablets
  • 🌐 Browser: Preferred for flexibility but concerns about performance limitations
  • 🔄 Cross-Device: Users frequently switch devices and locations for collaboration and different tasks
Phase 3

Tasks Don't Differ, They Scale

A crucial insight emerged: users wanted to perform the same types of tasks on all devices, but the complexity and duration of these tasks scaled based on device mobility.

Task Rationalization: Consumption vs Creation

Consumption Tasks: Analyze documents, view markups, analyze 3D models, view tasks from project manager

Creation Tasks: Create 3D models, create renderings, create toolpaths, adjust 3D models

Insight: Consumption tasks are naturally more mobile-friendly than creation tasks, informing our progressive disclosure strategy.

Design Solutions

Based on our research insights, I developed comprehensive design strategies addressing three critical areas that needed rethinking for device independence.

1. Information Density

Adapting the interface for smaller displays while maintaining functionality through progressive disclosure and direct manipulation.

  • Smaller Displays & Larger Targets: Mobile devices require larger touch targets and display less information at once
  • Progressive Disclosure: Creating a less cluttered interface while providing access to necessary features when needed
  • Direct Manipulation: Allowing users to interact directly with objects in the canvas
  • Single Window Mode: Using single-document workflows or letting the browser handle tabs

2. Input Methods

Addressing new ergonomics, missing inputs, and feedback limitations across different devices.

  • New Ergonomics: Commands placed near edges for easier thumb access on tablets
  • Rethinking Missing Interactions: Multi-select enabled by default when no keyboard detected
  • Prioritized UI Feedback: Developing UI in separate threads for more responsive feedback

3. Data Assumptions

Rethinking our save model and data handling for mobile and web contexts.

  • Rethinking Save Model: Mobile and web users expect "always-save" functionality
  • Fluid Resiliency: Designing for less stable connections and varying data costs
  • Minimizing Large Transfers: Optimizing data usage for mobile contexts

Responsive Toolbar Positioning

Created a condensed toolbar version with device-specific positioning:

  • Desktop & Web: Top placement (familiar)
  • Tablet: Left placement (thumb-accessible)
  • Mobile: Bottom placement (thumb-optimized)

Deep Dive: Navigation Toolbar & Views Feature

During user testing of our surface prototype, several users suggested moving the navigation bar to the top-right corner next to the view cube. This feedback prompted a comprehensive research and redesign effort.

Research Process

Card Sorting Study

Conducted unmoderated closed card sorting with 22 users across 4 personas: Design, Manufacturing, Engineering, and Electronics.

Problems Identified

  • Navigation bar contained tools not relevant to navigation
  • Inconsistent navigation bars across workspaces
  • Hard to access, discoverability issues
  • Taking up too much vertical space
  • Bottom-center placement not ideal for smaller screens
  • Not scalable across devices
  • Not aligned with similar tools in other software

Information Architecture Approach

I used a multifaceted approach to tool placement, balancing frequency of use, hierarchy, existing user habits, and new user friendliness.

Usage Frequency

Analytics revealed rarely-used commands that could move to preferences

Customer Expectations

Card sorting revealed users' mental models for tool organization

Hierarchy Analysis

Categorized application-level vs. document-specific vs. workspace commands

New User Consideration

Ensuring approachability for users unfamiliar with the system

Competitive Analysis & Results

Research revealed that leading CAD and 3D tools place navigation controls near the ViewCube, typically in the upper right on desktop, web, and tablet. This validated our direction and aligned with user mental models from other similar software.

Design Evolution

Before: Cluttered bottom-center navigation with mixed functionality

After: Clean, modern navigation in top-right with improved IA:

  • Multiple views moved to Views menu
  • Canvas display settings in toolbar
  • Redundant options removed
  • Logical groupings with labels
  • Collapsible for space-saving

Project Management & Collaboration

Managing this complex, multi-year project required innovative approaches:

  • Asynchronous Stakeholder Engagement: Created recorded presentations linked to Mural boards for global feedback collection
  • Comprehensive Project Tracking: Built Airtable system connecting features, tasks, research, and feedback
  • Orchestra Conductor Role: Balanced core design with coordination across multiple UX teams and POC projects

Strategic Leadership Impact

Led cross-functional collaboration across UX teams, established async feedback systems via Mural and Airtable, and balanced current product needs with future architecture requirements - demonstrating systems thinking at enterprise scale.

Key Learnings

This multi-year project provided valuable insights into enterprise-scale UX architecture and cross-platform design strategy.

01

Tasks Scale, Don't Change

Users want to perform the same core tasks on all devices, but complexity scales down as mobility increases. This insight drove our progressive disclosure strategy.

02

Mental Models Matter

Balancing existing user habits with new interaction patterns requires careful research. Card sorting and competitive analysis were crucial for navigation decisions.

03

Async Collaboration at Scale

Managing global stakeholders requires creative communication strategies. Recorded presentations + Mural boards enabled meaningful async feedback collection.

04

Architecture Enables Innovation

Strong information architecture and systematic component thinking create foundations that accelerate future product development and maintain consistency.

05

Prototype Early and Often

The Surface prototype allowed us to validate architectural decisions and interaction patterns before full development, saving time and ensuring user satisfaction.

06

Cross-Platform Thinking

Designing for device independence required fundamentally rethinking assumptions about input methods, display constraints, and user contexts.

Project Status

While I wasn't able to see this product launched during my tenure, the foundation and architecture I established should be released by the end of 2025. The comprehensive research, testing, and design work provides a solid foundation for Autodesk Fusion's device-independent future.
